Part 1: Overall Platform Vision

What Going Balls Is

Going Balls — Web Edition is a 3D physics-based rolling platformer that lives entirely in the browser. The player controls a ball that rolls through procedurally generated (or community-created) obstacle courses, collecting coins, avoiding hazards, and unlocking cosmetic skins — all rendered with Three.js and driven by cannon-es physics.

It is designed as a mobile-first, installable PWA that works offline after first load, with optional real-time multiplayer features powered by WebsimSocket.

Core Design Pillars

  1. Physics-First Gameplay — cannon-es handles gravity, friction, restitution, and collision. The player steers the ball, creating a momentum-based flow state.
  2. Procedural + Community Content — 40+ segment types, 9 difficulty tiers, seeded PRNG. Players build tracks with 25+ parts and share them.
  3. Deep Progression — 70+ skins with abilities, 12 skies with gameplay conditions, powerups, builder XP.
  4. Resilient & Graceful — Fallback-everything. Degrades from multiplayer to fully offline.

Key Technical Decisions

DecisionRationale
Zero build stepES modules via import map from esm.sh CDN. No bundler.
DI via game objectEvery module function takes game as first arg. No globals.
CDN dependenciesThree.js r184, cannon-es, nipplejs from esm.sh. No node_modules in prod.
SHA cache bustingCI stamps git SHA into sw.js on deploy.
Fallback-everythingTextures→grey, GLTFs→geometry, sky→color, network→offline.

Part 2: Build Prompts

21 self-contained, copy-pasteable prompts to build every system from scratch.

Prompt 1: Project Scaffolding

Create a zero-build-step browser game "going-balls" with:
- package.json: devDeps only (three, eslint, vitest, lhci, serve)

No bundler. ES modules only.

Prompt 2: HTML Shell & Import Map

Create index.html with:
- Import map: three, cannon-es, nipplejs from esm.sh CDN
- UI: loading overlay, top-menu (HELP/SHOP/LEADERBOARD/SURVIVAL/BUILDER/COMMUNITY/MUSIC),
  joystick + jump button, #overlay, pointer lock hint
- SEO: OG tags, Twitter Card, manifest.json link
- Custom font (5x5dots.ttf), linked styles.css

Prompt 3: Three.js Scene

Create engine/scene.js exporting initScene(game):
- WebGLRenderer (antialias, sRGB, ACES, PCFSoft shadows)
- 11 shared materials (wood, finish, coin, pendulum, spinner, etc.)
- Sky system: inverted sphere, PMREM, crossfade transitions
- Ball skin: texture/color/emissive/gltf, GLB async swap
- disposeMesh() utility, textureCache

Prompt 4: Cannon-es Physics

Create src/physics.js:
- GRAVITY=-45, JUMP_FORCE=28, MAX_VELOCITY=22, STEER_SPEED=22
- Ball: Sphere(0.5), mass=100, angDamp=0.95, linDamp=0.5
- Input → steer force → velocity clamp
- Obstacle collision (pendulums, spinners, movers, meteors)
- Weather particles: rain, wind, fire sparks, heat shimmer, meteors
- Portal teleportation with particle burst

Prompt 5: Level Generation

Create src/levelgen.js:
- mulberry32 seeded PRNG, 9 difficulty tiers
- 40+ segment types (platform, ramp, pendulum, spinner, loop, spiral, etc.)
- createLevel(game, seed): segments = 15 + floor(level × 2.5)
- Even levels mirrored horizontally
- clearLevel(): dispose all geometry
- playCommunityTrack(): deserialize saved parts

Prompt 6: Ball Skin Database

Create src/ball_db.js with BALL_DB (65+ skins):
{ name, price, tex, type: 'texture'|'gltf'|'color'|'emissive',
  ability: { key: 'speed'|'jump'|'coins', base, perLevel },
  description }
Price-based speed bias up to +12% at 12000 coins.

Prompt 10: Animation Loop

Prompt 11: Speed Lines

Create src/speed_lines.js:

Prompt 12: Motion Blur

- 8 centre-weighted directional samples
- Activates above 60% of MAX_VELOCITY
- Half-float RT with fallback, resize handler
