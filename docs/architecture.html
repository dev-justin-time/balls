Quad-Core Architecture

Our engine leverages four distinct programming languages, each optimized for its specific domain.

JavaScript (Orchestrator)

Handles the DOM, Three.js rendering, and user input. Acts as the thin client.

Rust (Compute Muscle)

Compiled to WebAssembly. Handles physics, collision detection, and anti-cheat validation.

Python (AI & Parsing)

Runs as a local FastAPI microservice. Handles PDF parsing, AI wireframe generation, and LLM integration.

Lua (Logic & Scripting)

Embedded via WASM. Handles procedural generation rules, shop logic, and dynamic prompt engineering.

Security & Anti-Reverse Engineering

High-value logic is never exposed to the client. Physics constants are injected at runtime via hashed payloads, and the WASM binary utilizes control-flow flattening to frustrate decompilers.
