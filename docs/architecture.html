Going Balls вҖ” Quad-Core Architecture

Version 2.0.0-alpha вҖ” Multi-Language Physics Platform

Four languages, each running in its strongest environment, connected through a secure IPC bridge:

  • JS JavaScript вҖ” Thin client orchestrator, Three.js rendering, DOM UI, input handling
  • Rust Rust (WASM) вҖ” Obfuscated physics solver, anti-cheat velocity validation, chaotic constant derivation
  • Python Python (FastAPI) вҖ” Secure level generation, WASM secrets, frame validation, rate limiting
  • Lua Lua (wasmoon) вҖ” Game theory economy, decoy pricing, endowed progress, battle pass rules

Language Strengths

JS JavaScript вҖ” Orchestrator

Built-in strength: DOM manipulation, event-driven async, browser APIs, Three.js integration.

The JavaScript layer owns the Game class, initializes all sub-systems, manages user input, renders with Three.js, handles DOM updates, and routes requests to the other three language runtimes via the QuadCoreBridge.

Rust Rust (WASM) вҖ” Compute Muscle

Built-in strength: Zero-cost abstractions, memory safety, WASM compilation, control-flow obfuscation.

Compiles to WebAssembly and runs inside the browser. Handles all physics computation вҖ” velocity integration, gravity, friction, collision response вҖ” using server-injected chaotic constants that prevent memory scraping. Includes dead-code injection and opaque pointer structures to frustrate reverse engineering.

Python Python (FastAPI) вҖ” Secure Backend

Built-in strength: Rich ecosystem (FastAPI, cryptography, numpy), rapid backend development, procedural generation.

Runs as a separate server process. Generates level layouts server-side with deterministic RNG, encrypts them with Fernet, and serves them to the client. Provides WASM physics secrets, validates frame hashes, and enforces rate limiting.

Lua Lua (wasmoon) вҖ” Economy Engine

Built-in strength: Embeddable lightweight runtime, hot-reloadable game logic, sandboxed execution.

Runs inside the browser via wasmoon (Lua 5.4 VM compiled to WASM). Handles all monetization logic вҖ” decoy pricing calculation, endowed progress tracking, battle pass stamp math. Game logic can be updated without recompiling the JS application.

Recycled JavaScript Modules

The following modules from v1.x are reused directly without modification:

  • engine/scene.js вҖ” Three.js scene, camera, renderer, materials, sky
  • src/physics.js вҖ” cannon-es physics world (used as fallback)
  • src/rendering.js вҖ” rAF loop, camera follow, speed lines, motion blur
  • src/audio.js вҖ” Music, SFX, audio context, visualizer
  • src/persistence.js вҖ” localStorage save/load, RNG seeding, configs
  • src/ball_db.js вҖ” 70+ skin definitions
  • src/levelgen.js вҖ” Procedural level generation
  • src/speed_lines.js вҖ” Speed line VFX
  • src/motion_blur.js вҖ” Motion blur post-processing
  • src/notification_manager.js вҖ” Toast notification system
