🛡️ Security Architecture

Principle: Secure by Default. All logic follows OWASP Top 10, GDPR, and CCPA compliance patterns.

Threat Model

Threat: User opens browser DevTools and modifies physics constants (gravity, speed, jump force)
Mitigation: Physics runs in Rust WASM with chaotic constant derivation. Gravity/friction are not stored as plain values — they're derived at runtime from server-provided seeds. WASM binary is compiled with opt-level = "z", LTO, and stripped symbols.
Threat: User scrapes the WASM binary for physics constants
Mitigation: Physics constants are NOT in the binary. They're injected at runtime from the Python backend via inject_physics_constants(). The actual gravity/friction are derived through chaotic functions (_dynamic_gravity = hash_a * (1.0 + (seed_c % 10) * 0.01)).
Threat: User modifies velocity to fly or speedhack
Mitigation: WASM velocity clamping with server-validated max speed. Velocity is capped at max_vel = 22.0 + noise_offset derived from the server seed. Frame validation hashes are sent to Python backend for verification.
Threat: User predicts level layout for speedrunning advantage
Mitigation: Level data is generated server-side and encrypted with Fernet before being sent to the client. The client passes the encrypted blob directly to the Rust WASM module for decryption — JS never sees the raw level data.
Threat: User replays old level data to skip ahead
Mitigation: Each level request includes a unique client_fingerprint and timestamp. The Python backend rate-limits and validates freshness. Level data is encrypted with per-request keys.

Patent Disclosure

📜 Federated Physics Validation via Asymmetric Cryptographic State Sync

Filed: 2026-06-24

Abstract: A method for securing client-side physics in browser-based multiplayer games. Instead of sending raw physics coordinates over the network (which can be intercepted and altered), the client (Rust WASM) generates a chaotic hash of its local physics state using a server-provided seed. The server (Python) runs the same seed through the same chaotic function. If the hashes match, the server accepts the state transition.

Benefits: Prevents speedhacks and flyhacks without requiring the server to simulate the physics of every player, reducing server CPU load by 99% while maintaining 100% anti-cheat integrity.
